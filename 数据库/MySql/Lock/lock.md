# 数据库的锁
 ## 共享锁
 ```
 使用方式: select * from table where id = 1 lock in share mode;
 ```
 **效果**
 -  读取时允许多事务
 -  修改时候不允许
 
 **结论**
 
 一个事务已获取共享锁，当另一个事务尝试对具备共享锁的数据进行读操作时，可正常读；进行写操作时，会被共享锁排斥。因此从这个实验中可以得知：共享锁也具备排他性，会排斥其他尝试写的线程，当有线程尝试修改同一数据时会陷入阻塞，直至持有共享锁的事务结束才能继续执行
